Resetting the Construction

  • 每一次我们继承上一个prototype之后我们都需要reset其constructor = 其本身; 无需括号 因为这里不需要运行这个函数.
  • 所以每次设置其prototype之后 也设置其prototype一下

Calling the super Constructor

这种方法失败的原因是 在创建Circle的时候 我们因为使用的new关键字 所以这时候的this代表的是circle本身, 但是当我们调用Shape(color);方法的时候 其里面的this代表的是global或者window,因此不生效

如何解决这个问题呢? 我们要解决的就是让Shape中的this指向我们创建的对象 这时候 我们使用Shape.call 方法 完美解决

Intermediate Function Inheritance

为了解决两行代码比较丑 而且有代码重复使用的问题 所以我们把这两个方法封装到函数当中 这样我们就可以重复使用 这个方法就叫做Intermediate Function

Method Overriding

  • 函数的重写需要注意的是 要先继承重写 再去重写方法
  • 如果我们想调用父类的原型链方法 可以直接用Shape.prorotype.duplicate.call(this) 传入this是将目前的对象和Shape中的this绑定

继承 父类中Instance members

function extend(Child, Parent) {
    Child.prototype = Object.create(Parent.prototype);
    Child.prototype.construtor = Child;
}
function Shape() {
    this.color = 'red';
    this.talk = function () {
        console.log('Shape Talk');
    }
}
Shape.prototype.walk = function () {
    console.log('Shape Walk');
}
function Circle(radius) {
    Shape.call(this);
    this.radius = radius;
    function run() {
        console.log('Circle run');
    }
}
extend(Circle, Shape);
let c = new Circle();

只是原型链的继承的话 Shape中的Instance members 不会继承到子类中 如果想要Shape中的method需要在Circle方法中调用Shape.call(this); 这样的话 Shape的Instance members就都会继承了

Polymorphism

poly means many morph means form

多种 形态

意思就是 如果Circle继承自Shape 同时Square也继承自Shape 并且都重写了Shape中的方法

我们在调用不同对象的时候 不同对象触发与之对应的同名方法 这就叫做Polymorphism
