Prototype vs Instance Members

  • 问题: 如果我们用同一个构造方法创建了很多个对象 这些对象都有自己的方法 但是这个方法其实都是一样的 那么这样其实就浪费了很大的内存空间
  • 就像我们上节课所说的 Circle.prototype===c1.__proto__

我们可以随时在 parentBase 也就是大家共有的这个prototype池子中添加新的方法 这个新的方法不会被复制很多次 且只有一个 但是大家都可以用

所以综上所述 一个构造方法有 Instance members 就是创建之后每个对象都有的方法们 和 Prototype members 这个是大家公有的

重写prototype共享池里的方法

JS运行时 会先在这个对象的Instance members 里面去找 如果找不到 再去大家的共享池里面去找

就近原则 Instance members → prototypebase→ parentPrototypeBase (More Accessable)

Instance members 和 Prototype methods 之间是可以完全没问题的相互引用的!
