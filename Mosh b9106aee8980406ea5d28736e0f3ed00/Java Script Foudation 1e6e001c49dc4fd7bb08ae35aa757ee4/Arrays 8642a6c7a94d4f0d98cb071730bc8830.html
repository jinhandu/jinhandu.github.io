Finding Elements (Primitives) 匹配元素位置Value Type

查找元素有两种情况 Primitives and Reference Types

Notes:

  • indexof() 匹配元素所在的位置 (从前开始匹配)
  • lastIndexOf() 匹配元素所在的位置 (从后开始匹配)
  • includes() 数组中是否有该元素
const numbers=[1,2,1,3,1,4];

console.log(numbers.indexOf(1));
//匹配第一个结果
//如果没有找到返回-1 如果找到 返回位置

console.log(numbers.lastIndexOf(1));
//匹配最后一个结果
//如果没有找到返回-1 如果找到 返回位置

if(numbers.indexOf(1)!==-1){console.log('exsist');}
if(numbers.includes(1)){console.log('exsist simple');}
//返回结果不为-1 表示匹配成功
  • if(numbers.includes(1)){console.log('exsist simple');} 判断一个primitive是否存在的干净写法
  • 所有上述函数都可以添加第二个参数 第二个参数的意思是起始位置 从什么位置开始搜索

Finding Elements (Objects) 匹配元素位置 Reference Type

Notes:

  • find() 检测元素是否存在
  • findIndex() 检测元素所在位置

错误原因: 我们传入的对象和比较的对象其实是两个不同的对象

const courses=[
    {id:1,name:'a'},
    {id:2,name:'b'},
    {id:3,name:'c'}
];

const course=courses.find(function(course){
    return course.name==='b'&&course.id===2;
});

console.log(course);
  • Predicate Function 也叫做callback Function 回调函数
    • English
      A predicate function is a function that takes one value as input and returns true / false based on whether the value satisfies the condition. isEven() is a predicate function. Discover Functional JavaScript was named one of the best new Functional Programming books by BookAuthority!

    给一个输入参数 返回真/假

  • 找不到find方法的返回值是undefined

find工作原理:

find方法 会把数组里的每个数作为参数传入回调方法中 然后进行比较 如果找不到 则会比较下一个 如果找到 返回该对象 同时停止查找

findIndex

const course=courses.findIndex(function(course){
    return course.name==='b'&&course.id===2;
});
  • 返回值是对象所在的位置
  • 如果没有找到返回-1

Arrow Functions 箭头方法 简化Code

Notes:

  • 去掉function关键字
  • 如果只有返回值 去掉大括号和return
  • 去掉return最后的分号

使用场景

  • 如果我们想把一个函数做为参数传递给一个函数
  • 一个函数的callback method
const course=courses.find((course)=>{
    return course.name==='b'&&course.id===2;
});
  • 改进后的方法
const course=courses.find(course=>{
    return course.name==='b'&&course.id===2;
});
  • 只有一个参数 去掉括号
  • 如果不传参数 则要写一个()代替
const course=courses.find(course=>course.name==='b'&&course.id===2);
  • 如果只有一个返回值 则: 去掉大括号 去掉return 去掉最后的分号

Testing the Elements of an Array 检测数组中的元素们是否符合某条件

  • every() 是否全部符合条件
  • some() 是否至少有一个符合条件
  • every()检测是否数组中所有的元素都满足指定条件
  • 数组中多有的元素都会依次进入every的回调方法 如果发现不符合条件的则停止搜索 并且返回false
  • some method 检测至少有一个元素复合条件
  • 如果找到一个复合条件的元素 那么检测会停止 并且返回true

注意事项:

some和Every都是新方法 旧的浏览器不支持 如何支持新的浏览器后面会提到

  • map() 筛选把数组中的每个元素 放到一个由新对象或新字符串组成新的数组中

创建数组之后 使每一个数组中的元素对应一个<li>

  • map方法返回的是一个数组 所以数组中的元素是用','逗号隔开的
  • 然后这时候 我们需要用itmes.join();把它拼成无逗号的字符串
  • we can use map method to map each element in an Array to something else

  • 需要特别注意的是 这里如果我们简化箭头方法 我们如果想返回一个对象 那么这个对象需要用小括号括起来 否则系统会默认{}是代码块 而不是对象

Reducing an Array 数组累加

  • reduce() 对数组中的元素全部进行一遍操作 这个操作可以是累加
  • reduce method的第一个参数是一个回调方法, 第二个参数是给回调方法的第一个参数 也就是accmulator设置初始值的
  • 如果我们不给accmulator设置初始值 那么这个函数会把数组的第一个值作为初始值传递给accmulator 这样的话这个函数的运行次数会少一次
  • return的值会保存到accmulator中 在下次函数调用的时候使用
