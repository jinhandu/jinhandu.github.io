Hoisting

  • hoisting meaning

    Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution. Inevitably, this means that no matter where functions and variables are declared, they are moved to the top of their scope regardless of whether their scope is global or local.

    提升是一种JavaScript机制，在执行代码之前，将变量和函数声明移至其作用域的顶部。 不可避免地，这意味着无论在何处声明函数和变量，无论它们的作用域是全局的还是局部的，它们都将移至其作用域的顶部。

Hoisting is to move the function declarations to the top of the field

  • 用function declaration的时候 我们可以提前用这个方法 定义在后面 因为JS引擎会通过Hoisting机制把这个函数的declaration放到其作用域的最顶部
  • 如果我们用function expression 则一定要先定义再去使用

Let vs Var

  • if 后面只有一条语句 let定义的变量也会限制在这个语句当中
  • 使用var定义的变量 其作用域是函数scope let 定义的变量作用域是代码块
  • var 定义的全局变量会被放到window对象中 而成为其一个属性 let 不会
  • 我们平时定义的function其实也被添加到windows 变量中去了
  • 总结 不要使用var

The "this" keywords

  • mothod → obj
  • function → global(node)/window(Chrome)
  • 如果this是在一个对象的method当中 那么this代表这个对象本身
  • 如果this是在一个正常的函数当中 那么this代表的是global in Node window in Browser
  • 如果我们在一个对象method使用一个回调方法，那么这个回调方法并不是函数的method而且一个function 这时候一个this代表 window
  • 解决方案，可以将this作为参数传入回调方法

Changing the Value of ''this''

  • 三种方式解决回调方法里的this与method里的this 不一样
  • 第一种 设置一个常量const slef,that=this.
  • 第二种 使用.bind(this) 使前面函数中的this与对象中的this永久绑定
  • 第三种 使用ES6箭头方法 这个箭头方法从容器中继承了this的值 直接就能用this代表这个Object 了
  • 使用apply call 方法 第一个参数传入会让这个函数内的this指向这个对象
  • bind()方法会让this和第一个参数永久绑定 并且返回一个新的函数
